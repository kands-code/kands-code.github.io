## 计算机体系结构与操作系统

这章算是操作系统相关的基础知识，但是如果是之前从未接触过相关知识的人来说，这些东西都是全新的，而且多而杂，很难去记忆，算是块硬骨头

### 计算机科学与计算机系统

#### 计算科学

计算机科学，或者计算科学，CS，是一门控制复杂度的学科

> the techniques used to control the intellectual complexity of large software systems[^1]

对于计算机，书上说这样描述的

> 计算机本身是一个**软硬件结合**的复杂系统，用来进行信息的计算处理，存储和传输，从而解决现实世界中与 *计算* 相关的各种实际问题

这里的计算，或者说 *Computing*，是有别于我们日常的计算 (Calculation) 的

计算被定义为了: “**<mark>任何需要创建或使用计算机的活动</mark>**，包括为广泛的应用目的而设计和构建软硬件系统，处理、存储和管理各种类型的信息，使用计算机进行科学计算，使计算机系统智能地运行，创建和使用通信和娱乐媒体，查找和收集与特定目的相关的信息等，实际上这个列表是可能无限扩展的”

> 原文：
>
> Computing is any goal-oriented activity requiring, benefiting from, or creating computing machinery. It includes the study and experimentation of algorithmic processes and development of both hardware and software. It has scientific, engineering, mathematical, technological and social aspects. Major computing disciplines include computer engineering, computer science, cybersecurity, data science, information systems, information technology and software engineering.[^2]

可见计算本身是和计算机分不开的

人类进行科学研究，在计算机技术发展之前，都是依靠理论研究和科学实验的，但计算机发展之后，计算分析成为了第三种方式

即，<mark>科学研究有三种手段，*理论研究*，*科学实验* 和 *计算分析*</mark>

#### 图灵机与计算机系统

*图灵机* 是图灵 (Alan Mathison Turing, 1912-1954) 于1936年发表的《论数字计算在决断难题中的应用》论文中提出的 **理想模型**

图灵机由一条可向两端无限延伸且被划分为一个个小格子的纸带、一个控制器和一个可以在带子上左右移动的读写头组成

图灵机的所有状态数目是**有限的**，并且有个特殊的状态，停机 (halting) 状态

<mark>一切合理的计算模型都等同于图灵机</mark>，所以我们可以用图灵机模型去模拟估计一个实际的计算模型

意义:

- 证明了通用计算理论，肯定了计算机实现的可能性
- 引入了读写，算法与程序语言的概念
- 计算极限为通用图灵机的极限

真正实现计算机的是冯·诺依曼 (John vo Neumann, 1903-1957)，他的计算机结构被称为“冯·诺依曼结构”，除此之外，还有个哈佛结构，不过冯诺依曼结构是主流

冯·诺依曼结构由**控制器**，**运算器**，**存储器**，**输入设备**和**输出设备**五部分组成，控制器及其状态对应着CPU，即运算器和控制器部分，纸带对应着存储器，读写头对应着输入输出设备

存储器和CPU间的交互，即存储，对应着空间域，输入输出设备与CPU间的交互，即指令流，对应着时间域，冯·诺依曼结构的计算机很好地映射可现实世界的时空观

### 计算机体系结构的基本概念

#### 摩尔定律

每过18个月，芯片上可集成的晶体管数量将增加一倍

单核处理器频率随着晶体管数量增长而不断提升，多核处理器芯片功率和频率基本不再增长

#### 计算机系统的层次结构

0. 数字逻辑层
1. 微体系结构层
2. 传统机器层
3. 操作系统层
4. 汇编语言层
5. 高级语言层
6. 应用程序层

其中第0到第2层**主要由**硬件实现，第3到第6层**主要由**软件实现，但 **<mark>计算机系统的软硬件没有一个明确的分界面</mark>**，一般以指令集作为分界面

#### 指令集

指令集是一个计算机系统支持的所有**机器指令**的集合

指令集，又称为指令集体系统结构 (Instruction Set Architecture, ISA)，ISA主要被分为CISC和RISC两类，而X86指令集，即许多与早期X86指令集 (Intel 8086 CPU) 兼容的指令集，是被认为现存唯一的CISC指令集

ARM和MIPS都是RISC指令集，但**MIPS的内核寄存器比ARM多一倍**，在同样的性能下MIPS的功耗会比ARM更低

1. 指令格式

    主要的指令格式为 `<操作码> (<操作数>)`

    其中操作码指明操作的类型，操作数指明具体操作的数或者操作数的地址，根据操作数的数量，指令格式可分为

    - 零地址指令
    - 一地址指令
    - 二地址指令
    - 三地址指令
    - 多地址指令

    汇编通过 *助记符* 代表操作码，使得编写程序容易了一点，例如:

    ```assembly
    MOV AX,0010H
    MOV BX,0020H
    ADD AX,BX
    ```

2. 寻址方式

    寻址方式指如何确定指令中操作数的地址及下一条指令的地址

    由于采用了程序计数器 (PC)，因此对 *非转移指令*，寻找下一条指令的地址只需要对PC递增即可

    对于转移指令需要由操作数给出下条指令的地址

    按照指令的寻址空间来分，寻址方式可分为 **<mark>立即数寻址</mark>**，**<mark>寄存器寻址</mark>**，**<mark>主存寻址</mark>** 和 **<mark>堆栈寻址</mark>** 这4种

3. 指令的功能分类

    - 数据传送指令

        - 取数指令: LOAD
        - 存数指令: STORE
        - 存储器或寄存器之间数据传送指令： MOV

    - 算术运算指令

        - 加法: ADD
        - 减法: SUB
        - 乘法: MUL
        - 除法: DIV

    - 逻辑运算指令

        - 逻辑与: AND
        - 逻辑或: OR
        - 逻辑非: NOT

    - 程序控制指令

        - 转移指令: JUMP (无条件或有条件)
        - 程序调用指令: CALL
        - 程序返回指令: RETURN
        - 循环控制指令

    - 输入输出指令
    - 处理器控制和调试指令

        - 空操作指令: NOP
        - 复位中断指令: CLI
        - 复位进位标识指令: CLC

4. CISC与RISC

    CISC结构追求的目标是强化指令功能，减少程序的指令条数，以达到提高性能的目的

    RISC体系结构的基本思想是尽量简化计算机指令功能，之保留那些功能简单，能在一个 *节拍* 内执行完成的指令，而把复杂的功能用一段子程序来实现

    RISC通过减少指令种类，规范指令格式和简化寻址方式等方法，方便处理器内出的**并行处理**，提高超大规模集成电路 (VLSI) 器件的使用效率，从而大幅地提高处理器的性能

    **RISC技术在CPU芯片上设置大量寄存器，用来存常用的数据**

#### 存储系统

存储系统可根据其读取速度和价格可分为以下层次

1. 寄存器组，**是汇编程序员可编程的空间，也是高级语言中编译器可支配的空间**，寄存器见数据的传递只需要**一个时钟的周期**
2. 高速缓冲存储器 Cache，位于主存和CPU间
3. 主存储器，**是存储系统的核心**
4. 辅助存储器
5. 脱机存储器

存储系统必须能够向应用程序员提供足够大的程序空间，且*无需应用程序员参与调度*

在应用程序员心目中，整个存储系统是一个单一的，直接可寻址的主存，其速度接近于离CPU最近的存储层次，其容量和每位价格接近于离CPU最远的存储层次，这是因为 <mark>程序对存储空间的访问具有**局部性**</mark>

把空间位置邻近的信息作为一“块”或一“页”放到靠近CPU的一级存储器M1中，根据局部性原理，在之后的一段时间内的访存有很大可能会在M1的同一块或同一页中找到需要的信息，称之为**命中**

“Cache-主存-辅存”是存储体系里最重要的三级层次，通常分成“Cache-主存”和“主存-辅存”两个两级存储系统来讨论

“主存-辅存”两级存储系统也称为**虚拟存储系统**

1. Cache-主存

    为了便于进行地址的映射和变换以及替换控制需要把Cache和主存划分成**大小相同**的块

    **<mark>块是Cache与主存之间的信息传输单位</mark>**，其中Cache块也称为Cache行 (Cache Line)，**是Cache存取的基本单位**

    根据数据是否仅在某一Cache中，多级Cahce的工作方式可分为Inclusive (共享) 和Exclusive (独占)

    “Cahce-主存”层次主要涉及以下几个关键问题：

    1. 地址映射问题

        Cache和主存见最基本的地址映射方式有**全相联映射**，**直接映射**和**组相联映射**

        全相联映射，主存中的第i块可以放入Cache中的任意一个位置，缺点是线路复杂，成本高，速度低，只适用于小容量的Cahce

        直接映射，主存按照Cache大小划分成若干区，每个区域按照Cache的分块而分块，每块之间都一一对应，这样访问速度块，硬件设备简单，但会导致Cache频繁换出且命中率低，Cache空间利用率低

        组相联映射就是前两者的结合，或者说，前两者就是组相联映射的特例

        主存中的一个组内的块数与Cache的分组数量相同，组建采用直接映射，组内采用全相联映射

        全相联映射为M路组相联映射 (M为Cache的总块数)，直接映射是1路组相联映射

    2. 替换策略

        常用的替换算法有**随机法 (RAND)**，**先进先出 (FIFO)**，**最近最少使用法 (LRU)**

    3. 数据一致性

        一致性是指同一份数据在不同存储位置的多个拷贝应保持一致，Cache和主存的数据一致性是通过Cache的特定写操作方式来保证

        - 写直达 (Write Through)： 每次修改立刻更新到主存，常会引起 *总线事务的竞争*，一致性高，效率低
        - 写回 (Write Back): 修改后在特定时机才会更新回主存

        多核处理器中，处理器内核之间的缓存一致性通过 **<mark>缓存一致性协议</mark>** 来维护，最经典的协议是 *MESI*，表明Cache Line的四种状态，在MESI协议中，Cache的控制器不仅知道自己的读写操作，还会监听 (Snoop) 别的Cache的读写操作以更新自己的状态，以此来保证数据在多个Cache之间的一致性

2. 虚拟存储系统

    为避免程序在不同机器间运行时由于地址问题导致崩溃，系统需要为应用程序提供一个虚拟的存储空间，程序在编写是都采用虚拟存储空间的地址，运行时再将寻你存储空间映射 (map) 到实际的物理内存中

    虚拟存储系统的大小 (理论最大值) 是由总线决定的，一般在32位系统上，空间大小就是4GB ($2^{32}$B)
