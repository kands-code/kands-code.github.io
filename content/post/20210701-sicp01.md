为什么要学习 *SICP* 呢？主要是想了解一下计算机的一些基础知识，更多的是想要了解一下**Lisp**，或者说**Scheme**这门语言

SCIP，全称是 *Structure and Interpretation of Computer Programs*，从标题我们就可以知道，这是本教我们如何构建程序的书，但这本书和其他上手就教我们如何构建一个实在的程序的书不一样，这本书主要是介绍了一种思想，如何去构思一个程序，需要注意什么

## Environment

环境搭建，关于这本书的代码和实现基本上是使用 *Scheme* 语言的

> Scheme的哲学是：设计计算机语言不应该进行功能的堆砌，而应该尽可能减少弱点和限制，使剩下的功能显得必要。Scheme是第一个使用静态作用域的Lisp方言，也是第一个引入“干净宏”和第一类续延的编程语言。 [^1]

`MIT-Scheme`是他们实际使用的语言，所以使用这个就足够了，但是由于MIT-Scheme不支持Windows了，所以需要想一个办法，比较好的替代方案就是`Racket`这个语言，也是 *Scheme* 的一种实现，但是更加现代，支持很多新的特性，主要用于语言设计，可以设计一些 *DSL*，Domain-Specific Language

相较于传统的Scheme或Lisp，Racket的优势之一就是有`raco pkg`这样的工具支持，可以下载库和包，也就是支持语言拓展

对于SICP，我们既可以使用Racket语言，也可以通过

```shell
raco pkg install sicp
```

下载相应的语言支持

Racket需要指明你使用的是哪门语言，故在编写相关的Racket代码的时候需要使用`#lang`来指定语言

```racket
#lang racket ;; 使用Racket语言
```

或者使用SICP语言规范

```racket
#lang sicp ;; 使用与SICP配套的语言规范
```

比较推荐的包还有

- `racket-langserver` LSP后端，可以配合vscode或者(n)vim来使用
- `r5rs`, `r6sr`, `r7sr` 几种语言规范，详情可以参考这里：[r5rs](https://docs.racket-lang.org/r5rs/), [r6rs](https://docs.racket-lang.org/r6rs/), [r7rs](https://www.gnu.org/software/guile/manual/html_node/R7RS-Standard-Libraries.html)
- `plait` 一种Racket *方言*，主要用于教学

除此之外，在Linux和MacOS中，你还可以选择是guile，gnu实现的一个scheme，来完成本书的习题，基本上是足够的

## Chapter01 Building Abstractions with Procedures

这一章，从标题就可以得到两个很重要的信息，第一个是 *Abstraction*，要构建抽象，说明这一章讲的是如何抽象

第二个就是 *Procedure*，步骤，这与 *Process* 是区分的，在之后会讲到

正如书中所言，“We are about tp study the idea of a computational process”，这算是对本章最好的总结，我们要学习如何 *计算*

在第一小节的前面还提到了软件工程，软件工程师们需要能将软件的运行过程可视化 (visualize) 出来，并且需要知道怎样做才能避免一些意料之外的 （unanticipated) 不会导致程序崩溃，并且需要能够debug，修改errors (glitches, bugs)

### Programming in Lisp

本章是介绍Lisp的优势和特点，Lisp能够描述过程 (processes)，这就是为什么真本书选择了Lisp作为主要语言，但实际上使用的是`MIT-Scheme`，Scheme算是Lisp的一种实现，但是在一些细节上与Lisp有所不同，比如在Lisp的基本的7个表达式或操作中，有一个用于判断是否是原子表达式的操作或表达式，ATOM，但是在Scheme中没有实现这个方法，他们使用了另一种方式，详情可以参考[这里](http://hyperpolyglot.wikidot.com/lisp)

当然，你可以手动实现这个操作

```racket
(define (atom? x)
  (and (not (null? x))
       (not (pair? x))))
```

这段代码的具体含义我之后会提，现在不懂也没关系

### The Elements of Programming

这里开始介绍一些编程的具体概念，首先是编程的要素

- primitive expression

    原始表达式，即预先定义好的一些操作，其余操作都可以由这些原始表达式实现

- means of combination

    如何将这些要素结合起来，与要一种方法，或者说规范

- means of abstraction

    如何抽象，这是极为关键的，我们需要一种抽象的方法来抽象出一些操作以完成更复杂的事情

以上三点就是书中描述的编程的基本要素 (Elements)

#### Expressions

表达式，可以说就是一个值，在百度百科里是这样定义的

> 表达式，是由数字、算符、数字分组符号（括号）、自由变量和约束变量等以能求得数值的有意义排列方法所得的组合。约束变量在表达式中已被指定数值，而自由变量则可以在表达式之外另行指定数值。[^2]

在Scheme中，数字是一种原始表达式，如果你在Scheme的interpreter里输入一个数字，他会将对这个数字 *evaluate* 后得到的值返回，通常会原样输出

```
$ racket # 以racket为例
Welcome to Racket v8.1 [cs].
> 468
468
> 1.200
1.2
```

> 退出racket解释器，使用`,exit`

其中`$ `是Linux中shell的默认提示符 (prompt)，而`> `是racket的提示符

你还可以使用guile

```
$ guile
GNU Guile 2.2.7
Copyright (C) 1995-2019 Free Software Foundation, Inc.

Guile comes with ABSOLUTELY NO WARRANTY; for details type `,show w'.
This program is free software, and you are welcome to redistribute it
under certain conditions; type `,show c' for details.

Enter `,help' for help.
scheme@(guile-user)> 386
$1 = 386
scheme@(guile-user)> 0.1210
$2 = 0.121
```

> 退出guile使用`,q`

guile中`scheme@(guile-user)> `是他的默认提示符，下面输出前使用了`$<n>`这种格式说明是第几个输出

使用操作符号，在Scheme中，加减乘除都有很好定义，就是`+`, `-`, `*`, `/`，但是没有定义通常编程语言里会定义的取余

在Scheme里，或者说Lisp系里，一般的表达式之间结合 (combine) 的格式是`(<operator> <operands>)`，operands就是操作符operator要操作的对象
，可以是多个，也可以只是一个，例如:

```
> (+ 2 3)
5
> (- 2 3)
-1
> (* 2 3)
6
> (/ 2 3)
2/3
> (/ 2.0 3)
0.6666666666666666
```

注意到对于除法，如果两个操作数都是整数，那么将返回一个整数或分数，而一旦有一个操作数是浮点数，也就是小数，那么就会返回一个小数，精度为机器精度[^3]，Scheme中**整数**的长度**没有限制**

通过复合表达式，我们可以算出来一个值，我们可以将这个值作为一个operand参与运算

```racket
(+ (* 3
      (+ (* 2 3)
         (+ 3 5)))
   (+ (- 10 7)
      6))
```

对于比较复杂的复合表达式，括号比较多，不便于阅读，故可以使用缩进的方式使其变得较为美观，这叫做 *pretty-printing*

#### Naming and the Environment

一般而言，对于编程语言都有个 *变量* 的概念，Scheme中也有，变量的定义方式是`(define <nams> <value>)`，定义变量后我们就可以使用他们了，例如:

```racket
#lang sicp
(define radius 2)
(define pi 3.1415926)
(define areaSize (* pi radius radius))
(display areaSize) (newline)
```

这里，`(display <obj>)`用于在屏幕输出东西，而`(newline)`用于输出换行

我们定义了两个变量，`radius`和`pi`，之后使用这两个变量进行运算，将算出来的结果绑定到`areaSize`上，最后输出了这个值

这里，`(* pi radius radius)`还可以写作`(* pi (* radius radius))`

对于一个较复杂的程序，会有很多这样的绑定，或者说对应关系，要管理他们，我们可以使用环境 (Environment)，这在之后会提到
